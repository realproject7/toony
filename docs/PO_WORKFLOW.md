# PO Workflow

This repository follows a ticket-first workflow.

## Required Order

1. Open Design package
2. Product architecture
3. Schema, CLI, and editor information architecture
4. EPIC and sub-tickets
5. Ticket review
6. Implementation
7. Complex merged PR review batches
8. Original webtoon episode production test

## Public Repository Safety

This repository is public. Do not include credentials, tokens, API key tails,
private URLs, private project files, wallet keys, provider account details, or
unredacted logs in issues, pull requests, review comments, commits, or docs.

## Implementation Rules

- No mock, stub, placeholder, or temporary runtime code. <!-- no-stub-ignore: states the no-stub rule itself -->
- Use test fixtures only when they are explicitly named fixtures.
- Keep webtoon craft study as internal product research, not GitHub tickets.
- Do not copy specific webtoon characters, panels, compositions, dialogue, or
  distinctive scene structures.
- Prefer simple implementations that match established project patterns.
